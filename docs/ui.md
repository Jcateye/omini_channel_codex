# Omini Channel Web 前端页面草案（交付 UI 工程师）

> 基于 `docs/project-status-2026-2-17.md` 的现状与建议，目标是给出一版“AI Native + 可直接操控 + 可追踪过程”的完整前端页面蓝图。

## 1. 设计目标（简要）

- **AI Native 主导**：每个核心页面都要有 AI 助手入口，不是独立玩具页。
- **直接操控优先**：高频动作（发消息、改规则、看效果）1~3 步完成。
- **过程可见可回放**：关键自动化流程（Campaign/Journey/Agent）必须有过程页与执行轨迹。
- **运营友好**：尽量减少原始 JSON 编辑，优先结构化/可视化交互。

## 2. 全局信息架构（建议）

### 一级导航

1. **工作台**
2. **Leads & 会话**
3. **Campaign**
4. **Journey**
5. **Agent Center**（Prompts / Tools / Routing / Knowledge）
6. **Analytics & ROI**
7. **CRM 同步**
8. **设置**（组织、渠道、权限、审计）

### 全局固定区域

- 顶部全局搜索：Lead、Campaign、Journey、消息、工具。
- 右侧 AI Copilot 抽屉：跨页可唤起，支持“帮我完成任务”。
- 全局通知中心：任务完成、失败告警、审批提醒。

## 3. 页面清单与功能点（简版）

## 3.1 工作台（Home / Ops Console）

- 今日核心指标：线索新增、发送量、回复率、转化、ROI。
- 待处理事项：失败任务、待审批活动、异常 Journey。
- 快捷动作区：新建 Campaign、新建 Journey、导入线索、启动 AI 分析。
- AI 建议卡：系统自动给出“下一步动作建议”。

## 3.2 Leads & 会话中心

- Lead 列表：标签、阶段、评分、来源、最近互动。
- Lead 详情：时间线（消息/标签/阶段变化/触达记录）。
- 会话面板：支持人工接管与 Agent 切换。
- 快速动作：打标签、改阶段、加入 Campaign/Journey。

## 3.3 Campaign 页面

- 列表页：状态（草稿/排期/运行中/已结束）、触达规模、回复与转化表现。
- 创建页（向导式）：受众 -> 内容 -> 发送策略 -> 预览 -> 发布。
- 实时执行页（过程页）：发送进度、失败原因、重试队列、状态回传。
- 复盘页：结果总结 + AI 优化建议（人群/文案/时段）。

## 3.4 Journey 页面

- Journey 列表：按目标（激活、召回、转化）分类。
- Journey 编辑器：可视化节点编排（触发、条件、延时、动作）。
- Journey 运行监控（过程页）：节点转化漏斗、卡点、失败节点重放。
- 模板库：按行业与目标提供可复用模板。

## 3.5 Agent Center（AI Native 核心区）

- **Prompt Studio**：提示词版本管理、A/B 测试、效果对比。
- **Tool Hub**：工具注册、权限配置、调用日志。
- **Routing 策略页**：Agent 分流规则、人工接管阈值。
- **Knowledge 页面**：知识源管理、同步状态、检索测试。
- **Agent 调试台（过程页）**：查看一次完整推理与工具调用轨迹。

## 3.6 Analytics & ROI

- 总览页：渠道表现、Campaign 排名、Journey 转化、归因模型切换。
- 归因分析：first touch / last touch / linear 对比。
- 趋势与预警：异常波动提醒（成本、回复率、转化）。
- 洞察页：AI 自动生成“原因解释 + 可执行建议”。

## 3.7 CRM 同步

- 映射管理：字段映射、规则校验、样例预览。
- 同步任务页（过程页）：实时状态、失败记录、补偿重试。
- 数据对账页：平台数据与 CRM 数据差异对比。

## 3.8 设置与治理

- 渠道配置：BSP 接入、Webhook、发送策略。
- 权限与角色：运营、管理员、审计角色。
- 审计日志：关键动作可追踪（谁在什么时候改了什么）。
- 系统健康：队列堆积、任务延迟、失败率、告警规则。

## 4. AI Native 交互要求（给 UI 工程师）

- 每个一级页都保留“让 AI 帮我完成”入口（固定位置、统一样式）。
- AI 输出必须可“一键采纳”为可执行配置，并允许人工二次编辑。
- AI 操作必须可解释：显示建议来源、影响范围、预期结果。
- 高风险动作（批量触达、规则覆盖）需要二次确认与回滚入口。

## 5. 关键任务流（必须做成明显入口）

1. **创建并发布 Campaign**（10 分钟内完成）
2. **发现转化下降 -> 定位原因 -> 调整策略 -> 再发布**
3. **新增 Journey 并观察节点转化**
4. **Agent 回答质量异常 -> 调试 Prompt/Tool -> 验证上线**
5. **CRM 同步失败 -> 定位字段问题 -> 修复并重试**

## 6. 交付建议（简要）

- 先输出低保真线框：优先工作台、Campaign 过程页、Journey 过程页、Agent 调试台。
- 再出中保真：统一 AI 交互组件（AI 抽屉、建议卡、执行确认弹窗）。
- 最后补视觉规范：状态色、告警层级、过程日志的时间轴样式。
